Staging Deleted Files in Github

by EveryDayKenneth

Posted on Tuesday, February 20, 2018

The Last Day of Chinese New Years 2018

Welcome to the year of the Dog! Dog's are the best! Yes they are! And today, I'm going to share how to stage deleted files on github with the "git add" command.

Normally git add won't add deleted files to your staging area (to be committed), so when you run

git add . or git *

you don't actually stage any deleted files.

If you want git add to actually stage deleted files, then you have to use the --update or -u flags:

git add --update or git add -u

Additionally, you can also use the --all or -A flag, but if I remember correctly, that flag only stages deleted files starting with Git version 2.0...or something like that. Note also that the flag will also stage untracked files, unlike -u, which only updates already-tracked files:

git add --all or git add -A
